Anteproyecto:

Título => Aid & Travel
Descripción => Plataforma de viajes e intercambio de habilidades e idiomas
Funcionamiento general => Hay dos tipos de usuarios: Anfitriones y Viajeros.
Los primeros publican el alojamiento y las actividades a desempeñar.
Los segundos eligen el alojamiento que más les convenga, con la consiguiente confirmación de los primeros.
Tecnología de Z usada => 1. REXX => para crear usuarios, anunciar alojamientos y buscar anfitriones.
2. DB2 => para almacenar datos.
3. JCL => para cargar los REXX.
4. z/OS Comm Server => para establecer comunicaciones entre el anfitrión y el viajero por medio de un tipo de mensajería básica o API REST simulada.
5. Sysplex => para que la aplicación tenga mayor escalabilidad a través de la selección de varios países, regiones o categorías.
6. Utilidades z/OS => para cargar/descargar datos y backups y, para llevar un mantenimiento.
Ventajas => 1. Es original y con impacto social.
2. Es escalable por regiones, idiomas y tipos de ayuda.
3. Es fácil de explicar y entender por un jurado o empresa.
4. Es ideal para practicar una lógica de negocio compleja.
5. Permite aplicar perfectamente todos los módulos de z/OS.
